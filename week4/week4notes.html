8. Forms

- const form = document.forms.search;
- If the form had the same name as any properties or methods of the document.forms object, such as 'submit', for example, that property or method would be referenced instead of the form element.
- input.addEventListener('change', () => alert('changed'), false);
- the alert message 'changed' only appears if you actually change the value inside the search box, then move the cursor away from it
- event.preventDefault();
- stop the form from being submitted to that URL altogether by using the preventDefault() method
- const hero = {};
- create a local variable called hero and assign it to an empty object literal
- hero.name = form.heroName.value;
- create a name property based on the input field's value
- <label for='age'>Age:<input type='number' id='age' name='age' min=0 step=1></label>
- Number input fields also have optional 'min' and 'max' attributes that can be used to limit the input given. The 'step' attribute is used to specify how much the value changes by each click.
- Select drop-down lists can be used to select one or more options from a list of values. The 'multiple' attribute is required if more than one option is to be selected.
- A <textarea> element is used to enter long pieces of text over multiple lines such as a comment or blog post. They work in much the same way as input fields. We access them using the 'name' attribute, and use the value property to see what text was entered.
- <button type='reset'>Reset</button>
- 'reset', which will reset all the form fields to their initial settings.
